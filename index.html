Amlang

Intersection of programming language, database, and simulation system.

Amlang is a language system that reverses the bifurcation into programming language and database, permits self-leverage in both Amlang and the underlying Rust, and maintains operational agnosticism. A key principle of Amlang is the ability to both collapse and reproduce the context of data.

A 10,000 foot view of Amlang is as a graph database (triplestore) with nodes containing and interconnected by S-exp-like structures, called the Environment. Agents exist in, modify, and explore the Environment, while using local state to apply subjective "lenses" to the shared Environment. Amlang code is evaluated into a representation within the Environment and can itself be explored and modified; consider an Agent exploring the code that it's currently executing. Indeed, many important components from the base system in Rust are reified in the Environment, allowing Amlang to modify and extend the system it exists on top of.

w3wm

Tiling desktop manager for Windows

w3wm is a project dedicated to providing Windows 7, 8, and 10 with an experience similar to that of using i3 on Linux. Windows are automatically tiled, and users can use the keyboard to navigate between, move, resize, open, and close windows. Built from the ground up with native support for multi-monitor setups and DPI-aware processes, w3wm is a desktop manager fit for the modern user who potentially has multiple monitors of high resolution.

Tetrad Game Engine

3D game engine using an entity-component-system architecture

The Tetrad game engine is a (now abandoned) 3D game engine built using the entity-component-system architecture. It uses OpenGL for rendering, and strives to remain platform-independent. Some notable features built from scratch are:

  • Entity-Component-System framework
  • 3D Rendering system using OpenGL
  • GUI system (using the rendering system)

The project was being developed and tested on Windows and Linux. Nowadays, I would want to use Amlang to develop such a system, with the end result being a mixture of compiled Rust and interpreted Amlang.

Reliable Data-Transfer Protocol

User-mode implementation of reliable data-transfer protocol

This is an implementation of a library that provides an interface similar to that of the Berkeley sockets API, with the library itself providing reliable data transfer over UDP. While there are certainly similarities between the implemented protocol and TCP, there are a number of differences (for example, TCP uses cumulative ACKs, whereas the implemented protocol uses ACKs in a manner similar to that of Selective Repeat).
